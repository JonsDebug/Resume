How I Run Claude

ENGINEERED SYSTEM

My Claude setup isn't a chat window — it's an engineered system: a thin always-on rule layer, a catalog of on-demand skills, deterministic hooks at session boundaries, layered memory stores, and mechanical safety gates in front of anything that touches a live org. Every finished ticket feeds a learning loop that promotes lessons into gates, skills, or rules — the system gets stricter and smarter with every session.

hardens the system
STG-01 · Session start — automated
  • Hooks fire — context injection · environment resolve · folder scaffold · learning counter
  • Rule stack loads — one master playbook, a single source of truth
  • Memory recall — global index · project memory · org schema notes
STG-02 · Working the ticket
  • Skills auto-trigger — ~17 on-demand procedures, loaded only when the task matches
  • Local-first knowledge — notes + code mirrors before any live query
  • Subagents fan out — discovery plus adversarial verification of findings
STG-03 · Safety gates — mechanical
wall, not a step
  • Pre-deploy gate script — hard stops, not reminders
  • Read-only org access — queries yes · writes never
  • Human runs every deploy — Claude prepares, never executes

  • Docs generated — audience-shaped: exec · admin · developer

  • Lessons Learned — captured in real time, not retrospectively
STG-05 · Recursive learning

  • Human approval — gates every promotion, nothing lands silently
  • Routing litmus — gate > skill > rule
