Working Towards

Windows Log Ingestion & Detection Lab

1. Objective

The objective is to:

  • Build an SOC home lab to bridge red and blue team tactics.
  • Deploy a SIEM, ingest logs to a centralized log server, and create custom detection logic.

2. Lab Architecture

The environment consists of:

  • A sandboxed virtualized network to ensure safe execution of malware and attack scripts.
  • Endpoint 1: Ubuntu 22.04 running Splunk Enterprise utilizing cross-platform logging.
  • Endpoint 2: Windows 10 as a primary target.

3. Environment Setup

Configuration steps:

  • Configured Splunk on the Ubuntu VM.
  • Set up a receiving port on TCP 9997.
  • Installed Sysmon on Windows with SwiftOnSecurity's custom configuration to capture process creations (EventID 1) and network connections (Event ID 3).
  • Adjusted my local group policy to enable audit process creation and include CLI events.

4. Attack Simulation

Executed three attack scenarios:

  • Brute Force: Utilized a Python script (Github link(if seen early updating soon)) to attempt 100+ RDP logins against the Windows machine.
  • Credential Dumping: Executed procdump against lsass.exe to simulate memory harvesting.
  • Persistence: Created a malicious task in task scheduler and used PowerShell to execute a base64 encoded download.

5. Log Forwarding

Log forwarding configuration:

  • Deployed the Splunk Universal Forwarder on the Windows endpoint.
  • Configured the UF to monitor:
    • WinEventLog://Security
    • WinEventLog://Microsoft-Windows-Sysmon/Operational
    • WinEventLog://System
  • Configured the Linux VM to forward auth logs to Splunk.

6. Log Analysis

Analysis findings:

  • Once data was indexed, I began a search for all attack artifacts.
  • Identified Event ID 4625 spikes during brute forcing.
  • Used stats command to correlate Event ID 4688 with unusual parent processes.

7. Detection Rules

Custom detection rules created:

  • Added 8 custom detection rules.
  • Primary Example: index=windows sourcetype="XmlWinEventLog:Sysmon" EventCode=1

8. Lessons Learned

Key takeaways:

  • Need a bigger hard drive for all these virtual machines.
  • Regular Windows logging lacks in-depth information and Sysmon is important.
  • Manual log analysis is slow and there needs to be a way to map and automate specific parts of the workflow.
